## 【最终任务】基于文心大模型的 deepin 操作系统环境感知助手

### 1. 任务背景

随着 AI 与操作系统的深度融合，操作系统需要具备“环境感知”能力。传统的感知依靠进程名判断，而新一代 AI 系统则通过分析用户的实时操作行为，如窗口标题、屏幕内容、交互动作等，主动理解用户意图并提供辅助。

本任务要求开发者在 **deepin 25** 系统上开发一个原型应用。该应用需利用系统技术手段获取当前用户的工作场景，并调用 **文心大模型（ERNIE / PaddleOCR-VL）** 进行场景分析与意图识别，最终触发相应的 UI 引导或服务推荐。

- 技术标签：环境感知，意图识别，C++/Python
    
- 任务难度：进阶

### 2. 任务描述

开发者需实现一个名为 **"deepin Context AI"** 的演示程序，核心流程如下：

1. **场景数据采集**：
    
    - **基础方案**：利用 Linux 系统接口，如 D-Bus 信号、`wmctrl`、读取 `/proc` 信息等获取当前活动窗口的元数据。
        
    - **进阶方案（加分项）**：利用系统截屏接口捕获当前操作区域，通过 **GUI 多模态理解（如 PaddleOCR、PaddleOCR-VL）** 识别屏幕内容。比如：正在查看的 PDF 标题、代码编辑器里的报错信息等，可以和基础方案结合。
        
2. **AI 意图识别**：
    
    - 将采集到的元数据或视觉特征发送给 **文心大模型**，请求模型判断用户意图，例如：用户正在尝试解决一个编程 Bug、正在准备一份会议 PPT、或是在进行学术调研。
        
3. **主动服务触发**：
    
    - 根据 AI 的判断结果，在正在操作的应用附近展示对应的智能体入口，例如：检测到在写代码，弹出“代码优化助手”；检测到在写文档，弹出“润色助手”。这些助手可以使用提示词实现一些简单的功能，并提供结果查看界面。加分项是调用MCP、skills等本地工具，完成一些特定任务。
        

### 3. 技术要求

- **系统环境**：deepin 25。
    
- **模型支持**：调用飞桨文心大模型 API（erniebot SDK）或部署飞桨轻量化多模态模型。
    
- **开发语言**：推荐使用 Python 或 C++/Qt。
    
- **关键点**：意图判断的准确性、对 deepin 系统资源的占用控制、以及 UI 与系统界面的融合度。
