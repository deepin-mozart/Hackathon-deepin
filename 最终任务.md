# 【打卡任务】deepin Agent Teams 

## 任务背景

deepin（深度）操作系统作为国内领先的Linux发行版，致力于为用户提供美观、易用、安全的桌面环境。随着人工智能技术的快速发展，将智能体技术融入操作系统，打造智能化的桌面体验成为可能。本次大赛以“deepin Agent Teams”为主题，邀请开发者构建运行在deepin操作系统上的多智能体协作系统，通过智能体技术提升用户的操作效率与体验。

- 技术标签：环境感知，意图识别，多智能体，MCP, Skills
- 任务难度：进阶

## 任务目标

设计并实现一个运行在deepin操作系统上的智能体应用——**deepin Agent Teams**，该应用具备“环境感知”能力，通过分析用户的实时操作行为，如窗口标题、屏幕内容、交互动作等，主动理解用户意图并调用相应智能体提供辅助。

该系统需满足以下核心要求：
### 1. 技术要求

- **系统环境**：deepin 25。
    
- **模型支持**：至少调用一款飞桨文心大模型 API（erniebot SDK）。
    
- **开发语言**：推荐使用 Python 或 C++/Qt。

- **隐私安全**：系统感知功能需注意用户隐私保护，敏感操作需获得用户明确授权。

- **交互入口：**
	- 提供唯一的用户交互界面（可以是悬浮球、侧边栏或独立窗口）
	- 用户通过自然语言输入任务指令
	- 系统需具备**智能决策调度**能力，根据用户指令自动选择合适的智能体执行

- **关键点**：基于环境感知的意图判断准确性，以及智能体调用的准确性。

鼓励使用deepin操作系统的特色功能（如控制中心API、DDE桌面环境特性），环境感知需注意对 deepin 系统资源的占用的控制。

### 2. 智能体能力要求

#### 2.1 基础能力
- **聊天对话**：具备基本的对话交互能力，能够回答常规问题

#### 2.2 工具使用能力
智能体需支持多种工具调用，包括单不限于：

| 工具类型 | 具体要求 | 示例场景 |
|---------|---------|---------|
| **系统工具** | Bash命令执行、文件搜索、应用启动/停止 | 查找文件、安装软件、启动应用 |
| **系统配置工具** | 修改系统设置（对标deepin控制中心） | 修改网络设置、调整显示、更改主题 |
| **MCP工具** | 支持MCP协议的工具接入 | 接入第三方服务、自定义插件 |
| **SKILLS** | 预定义的技能模块 | 邮件撰写、日程安排、信息汇总 |

#### 2.3 多智能体协同
- 构建至少包含3个不同职能的智能体团队（如：系统操作员、信息收集员、内容创作员等）
- 支持复杂任务的**自动拆解**与**分工协作**
- 智能体间可进行任务交接与信息共享

#### 2.4 系统感知能力（核心功能）
利用 Linux 系统接口，如系统 API、D-Bus 信号、`wmctrl`、读取 `/proc` 信息等获取当前活动窗口元数据、剪切板数据等。利用系统截屏接口捕获当前操作区域，通过 **GUI 多模态理解（如 PaddleOCR、PaddleOCR-VL）** 识别屏幕内容。比如：正在查看的 PDF 标题、代码编辑器里的报错信息等。

以下列举部分（仅做参考）智能体能够感知操作系统中的用户行为：

| 感知维度     | 具体要求            | 用途          |
| -------- | --------------- | ----------- |
| **鼠标操作** | 监听鼠标点击、移动轨迹     | 分析用户关注区域    |
| **键盘输入** | 感知键盘输入（需隐私保护机制） | 理解用户正在输入的内容 |
| **剪贴板**  | 监控剪贴板内容变化       | 获取用户复制的信息   |
| **应用状态** | 感知应用启动、关闭、窗口切换  | 了解用户工作上下文   |
| **屏幕图像** | 定时、连续或触发式截取屏幕   | 视觉理解用户当前操作  |

基于系统感知，智能体需能**分析用户意图**，主动提供自动化操作或相关信息。

### 3. 场景实现要求

参赛作品必须能够演示以下两个核心场景：

#### 场景一：智能邮件助手
当系统感知到用户意图发送邮件时（例如用户说出“给张三发邮件说项目进度”或在邮件客户端中开始撰写），智能体团队需：
1. 识别邮件主题与收件人
2. 自动从以下渠道收集信息：
   - 文件系统中搜索相关项目文档
   - 从网络中获取最新项目动态（需用户授权）
   - 从剪贴板中获取用户最近复制的内容
3. 整合信息生成邮件正文
4. 在邮件客户端中呈现（或询问用户后发送）

#### 场景二：系统问题自动诊断与修复
当用户输入系统相关问题（如“打印机连不上”、“没有声音了”、“帮我安装微信”等），智能体团队需：
1. 分析问题类型与可能原因
2. 结合系统感知获取当前系统状态：
   - 检查打印机服务状态/驱动情况
   - 检查音频设备与音量设置
   - 检查软件源与应用安装状态
3. 自动执行修复操作：
   - 重启服务、更新驱动
   - 调整系统配置
   - 执行安装命令
4. 向用户反馈处理结果

## 评分标准

| 评分维度       | 权重  | 具体要求                                |
| ---------- | --- | ----------------------------------- |
| **系统感知能力** | 30% | 感知的准确性、意图分析的智能化程度。将基于常见的意图触发场景进行打分。 |
| **功能完整性**  | 30% | 是否满足所有基础要求，场景是否完整实现                 |
| **智能调度效果** | 20% | 任务拆解合理性、智能体选择的准确性                   |
| **创新性**    | 10% | 在满足基本要求基础上的创新设计                     |
| **用户体验**   | 10% | 交互友好性、响应速度、结果展示                     |
